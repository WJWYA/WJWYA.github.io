解决vue cli3中axios调用接口的跨域问题
2020-12-13
vue 跨域 同源策略
技术

  在vue项目中使用axios调用api接口时，有时会在打印界面报错，请求的数据没有渲染在页面上，刷新几次又能回复正常，一直很奇怪不知道原因，百度查了一下才知道是跨域问题。那什么是跨域呢？

跨域&同源策略

  跨域，是指浏览器不能执行其他网站的脚本。通过阅读了一些资料我知道了导致跨域的罪魁祸首是浏览器同源策略，同源策略可以理解为浏览器的一种安全机制，所以跨域问题就是浏览器在搞鬼。那什么情况下会导致跨域呢？官方说如果两个 URL 的 protocol、port (如果有指定的话)和 host 都相同的话，则这两个 URL 是同源。,那意味着三者有一个不同，就会导致跨域问题。举个例子：
例如下表是与URLhttps://wjwya.top/webmusicplayer.github.io/dist/#/discovery的源进行对比：

URL 结果 原因
https://wjwya.top/webmusicplayer.github.io/dist/#/playlists 同源 只有路径不同
https://wjwya.top/WangY.github.io/ 同源 只有路径不同
http://wjwya.top/webmusicplayer.github.io/dist/#/playlists 跨域 协议不同
https://wjwya.top:81/webmusicplayer.github.io/dist/#/songs 跨域 端口不同
https://wjwzj.top/webmusicplayer.github.io/dist/#/songs 跨域 主机不同

  浏览器一般是针对接口的请求和Dom的查询来实现同源策略。

了解完大概的基础背景之后，下面来说一下我遇到的问题，由于报错的时候我没有截图，于是从网上找了相似的报错截图如下：


  于是相关资料后使用了较为简单的设置代理进行跨域，然后在开发环境（本地）测试发现完美的解决了跨域问题，打包上线后全**蹦了，一个接口都没调用出来，没记错的话是报的404错误。折腾了好久终于找到了解决生产环境（服务器）的跨域问题。

解决开发环境（本地）跨域问题

  之前在该项目打包上线时，我已经在package.json同级目录下创建了一个叫vue.config.js的文件（见：vue项目发布、更新到github），在vue.config.js文件中输入以下代码：

devServer:{

    proxy:{
     '/api':{                  //对 /api开头的接口进行代理
        target:"要代理的地址",   //API服务器的地址,例如为https://autumnfish.cn
        ws:true,               //代理websockets
        changeOrigin:true,     //虚拟的站点需要更换origin
        pathRewrite:{          //重写路径
          '^/api':''
        }
      }
    }
  }

在main.js中输入一下代码：

import axios from 'axios'
Vue.prototype.$axios = axios      //在只有一个代理地址的情况下可以设置统一的baseURL
axios.defaults.baseURL =  '/api' //这样之后所有的接口以代理地址开头的部分全部省略

举个例子：

axios({
    url:"https://autumufish.cn/banner",
    method:"get",
}).then(res=>{
    console.log(res)
})
//就写成
axios({
    url:"/banner",
    method:"get",
}).then(res=>{
    console.log(res)
})
//省略掉https://autumufish.cn

  此时已经解决了开发环境（本地）的跨域问题啦~
  通过浏览器我们可以查看请求的地址为https://localhost:8080/api/banner,但实际上请求的url为https://autumufish.cn/banner。

解决生产环境（服务器）跨域问题(好像有点问题)

  虽然此时已经解决了在开发环境（本地）的跨域问题，但当把项目发布到服务器端使，所有的接口均报错404的错误，为解决该问题，仅需修改main.js,修改一条语句即可：

Vue.prototype.$axios = axios      //在只有一个代理地址的情况下可以设置统一的baseURL
//axios.defaults.baseURL =  '/api' //这样之后所有的接口以代理地址开头的部分全部省略
axios.defaults.baseURL = process.env.NODE_ENV === "production" ? "https://autumnfish.cn" :  '/api'//该语句判断当前如果是在开发环境下，就将baseURL设置为后面的地址，如果不是，就继续使用设置好的api代理

  此时已经解决了生产环境（服务器）的跨域问题啦~
